# 자동 넘버링 및 비트시트 양방향 동기화 구현 계획

## 1. 회차 자동 넘버링 (Auto-numbering)
**문제점:** 현재 회차 제목에 "1화", "2화"를 수동으로 입력해야 해서, 순서가 변경되거나 회차가 삭제될 때 일일이 수정해야 하는 번거로움이 있습니다.
**해결 방안:**
- **UI 렌더링 변경:** 좌측 목록 및 에디터 상단에 회차 제목을 표시할 때, `type === 'chapter'` 인 회차들의 순서를 계산하여 자동으로 `1화 `, `2화 ` 접두사를 붙여 보여줍니다.
- **제목 입력 방식 변경:** 새 회차 추가 시 "1화" 같은 숫자를 쓰지 않고, 진짜 제목만 입력하도록 안내합니다.
- **기존 데이터 마이그레이션:** 기존에 수동으로 "1화 제목", "제2화 제목" 이라고 적혀있던 제목들에서 정규식을 이용해 앞의 "N화" 부분을 자동으로 제거하여 깔끔하게 정리합니다.
- **비트시트 매칭 연동:** 이미 비트시트 매칭 로직은 순서 기반(1번째 챕터 = 1화) 매칭을 지원하므로, 제목에 "1화"가 없어도 기획의 `## 1화`는 자동으로 1번째 챕터에 정확히 꽂히게 됩니다.

## 2. 비트시트 양방향 동기화 (Two-way Sync)
**문제점:** 작품 기획 탭에서 '배포하기'를 누르면 개별 회차로 비트시트가 들어가지만, 반대로 개별 회차에서 비트시트 내용을 수정했을 때 작품 기획 탭의 원본 비트시트에는 반영되지 않아 데이터가 파편화됩니다.
**해결 방안:**
- 개별 회차의 기획란(`planEditor`)에서 내용이 수정될 때, 텍스트 안에 존재하는 `### 📋 비트시트 — [헤더]` 블록을 감지합니다.
- 감지된 헤더(예: `1화`)와 수정된 내용을 바탕으로, '작품 기획' 탭에 있는 전체 비트시트 본문(`planSections`) 내의 `## 1화` 블록을 찾아 자동으로 내용을 교체(Update)해줍니다.
- 이 동기화는 사용자가 기획란 텍스트박스에서 포커스를 벗어날 때(onblur) 또는 타이핑이 멈춘 후(debounce) 자동으로 백그라운드에서 실행되며 서버에 함께 저장됩니다.

> [!IMPORTANT]
> **User Review Required**
> 1. 회차 제목에서 기존에 쓰셨던 "N화" 부분은 제가 일괄적으로 제거해 드려도 괜찮을까요? (예: "1화 아무 역할도 하지 않아도 됩니다" -> "아무 역할도 하지 않아도 됩니다" 로 변경되며, 화면에는 자동으로 계산된 `1화`가 앞에 붙어서 보여지게 됩니다.)
> 2. 프롤로그와 에필로그는 번호 매기기에서 제외되는 것이 맞지요?

## Verification Plan
1. 기존 데이터에서 "N화"가 정상적으로 제거되는지 확인.
2. 회차 순서를 드래그 앤 드롭으로 바꿨을 때 번호가 즉시 업데이트되는지 확인.
3. 개별 회차의 기획란에서 비트시트를 수정한 후, 작품 기획 탭으로 돌아갔을 때 전체 비트시트에 반영되어 있는지 확인.
